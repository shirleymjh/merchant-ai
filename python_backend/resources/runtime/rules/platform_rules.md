# yshopping 平台商家规则

## 基础服务

- yshopping 商家助手用于帮助商家理解经营数据、订单、退货、客服工单、理赔、优惠券、商品管理、保证金、申诉、身份信息等问题。
- 商家数据查询默认限定当前登录商家，当前本地商家 id 为 `100`。
- 涉及金额时默认展示人民币元；Doris 明细表中部分金额字段为分，需要在话术里说明换算或使用画像表元口径。

## 商品与上架

- 商品上架需完成商品信息补全、商品审核、风险审核等环节。
- 商品类目需与实际售卖商品一致，不可错放、混放或使用无关类目规避审核。
- 不同类目通常会关联不同的品牌资质、商品属性、规格信息和图片要求；发布前应先确认类目是否选对，再补齐对应资料。
- 商品标题、主图、详情页、视频、规格参数应与商品实际属性一致，不得夸大宣传、标题堆词或图文不符。
- 若商品为品牌商品、特殊经营商品或需准入类目，应先补齐品牌授权、资质证明或行业要求资料，再提交审核。
- 若商品审核被拒绝，应优先查看拒绝原因、商品类目、品牌资质、图文描述、货号和风控备注。
- 若商家咨询“类目规范”“类目要求”“类目怎么选”，应重点回答类目一致性、类目对应资质和错放类目的审核风险。
- 若商家咨询“上架要求”“怎么上架”，应重点回答资料补全、审核流程、风险审核和常见拒绝原因。

常见测试问法：
- 商品上架要求是什么
- 商品类目规范是什么
- 商品类目怎么选
- 上架商品需要哪些资质
- 商品审核被拒怎么办
- 图文描述有哪些要求

## 交易与发货

- 交易指标优先使用 `ads_merchant_profile` 的画像指标。
- 明细查询使用 `dwm_trade_order_detail_di`，按商家/卖家 id 过滤。
- 发货超时、签收超时等履约问题可结合订单明细与供应链明细分析。
- 商家发货时应确保订单状态、发货时效、物流单号和承运信息一致，避免虚假发货、超时发货或单号无轨迹。
- 若出现揽收超时、运输异常、签收异常等问题，应优先结合订单状态、物流轨迹和供应链履约环节定位原因。
- 若用户咨询“物流单号关联订单”，应优先说明可通过订单明细和物流信息核对订单号、发货状态和物流轨迹。
- 若用户咨询“发货规则”“超时发货怎么办”，应重点回答发货时效、物流信息准确性和异常订单排查路径。

常见测试问法：
- 发货规则有哪些
- 超时发货怎么处理
- 物流单号怎么关联订单
- 订单为什么一直未签收
- 签收异常怎么排查

## 退货与退款

- 退货指标优先使用 `ads_merchant_profile` 的退货、退款指标。
- 明细查询使用 `dwm_trade_refund_detail_di`，按商家/卖家 id 过滤。
- 直接退款、卖家责任退货、退款金额等问题需要同时给出数据口径和建议。
- 退款场景需先区分直接退款、退货退款、卖家责任退货、仅退款等不同类型，再判断处理口径。
- 商家处理退款时应重点关注订单状态、退款原因、售后凭证、责任归属和当前处理节点。
- 若商家咨询“退款规则”或“退款怎么处理”，应先解释口径，再说明需要查看退款原因、责任判定和售后状态。
- 若商家咨询“卖家责任退货”或“退款风险”，应强调高频原因排查，如商品描述不符、发货异常、售后响应不及时等。

常见测试问法：
- 退款处理规则是什么
- 直接退款是什么意思
- 卖家责任退货怎么判断
- 退款金额为什么变高
- 退款风险怎么排查

## 客服工单

- 工单指标优先使用 `ads_merchant_profile`。
- 工单明细使用 `dwm_cs_ticket_detail_di`，必要时结合 `ads_risk_csr_ticket_all_content` 的对话内容。
- 工单二次开启、催单、评价分下降时，应建议商家检查响应时效、物流进度和售后口径。
- 若工单量异常上升，应优先排查催单、物流异常、退款争议、商品说明不清或客服响应不及时等问题。
- 若商家咨询“工单进度”“工单状态”，应优先说明待处理、处理中、已关闭、二次开启等状态的业务含义。
- 若商家咨询“为什么工单被二次开启”，应优先排查处理结果未解决、回复不完整、物流未更新或售后争议未闭环。

常见测试问法：
- 工单规则是什么
- 工单进度怎么查
- 工单二次开启是什么意思
- 催单太多怎么办
- 评价分下降怎么处理

## 理赔与赔付

- 赔付指标优先使用 `ads_merchant_profile`。
- 明细使用 `dwm_cs_repay_detail_df`。
- 话术需要区分赔付订单数、赔付金额、赔付方式和赔付原因。
- 赔付问题通常需要先明确责任归属，再区分是履约问题、售后争议、商品问题还是服务问题导致的赔付。
- 若商家咨询“赔付规则”“赔付款为什么增加”，应说明赔付原因、赔付方式、金额口径及后续申诉路径。

常见测试问法：
- 赔付规则是什么
- 赔付款为什么增加了
- 赔付原因怎么查
- 赔付可以申诉吗

## 优惠券

- 优惠券指标优先使用 `ads_merchant_profile` 的支付成功优惠、交易成功优惠等字段。
- 明细使用 `dwm_coupon_detail_di`，按 `seller_id` 过滤。
- 优惠券问题需区分券模板、发放状态、适用商品、使用门槛、有效期和实际核销效果。
- 若商家咨询“优惠券规则”或“优惠金额怎么算”，应优先说明活动门槛、优惠口径和核销结果差异。

常见测试问法：
- 优惠券规则有哪些
- 优惠金额怎么算
- 为什么优惠券用了但成交没起来
- 优惠券明细怎么查

## 保证金、申诉与商家资料

- 保证金和商家信息优先查看 `dim_merchant_df` 与 `ads_merchant_profile`。
- 保证金充值明细使用 `dwd_merchant_deposit_recharge_df`。
- 申诉明细使用 `dwd_merchant_appeal_detail_df`。
- 若商家咨询保证金，应区分保证金余额、冻结保证金、初始保证金和充值记录。
- 若商家咨询申诉，应优先说明申诉原因、当前状态、补充材料和处理结果。
- 若商家咨询费率、结算、商家资料，应优先说明当前资料项、变更限制和需要核对的信息。

常见测试问法：
- 保证金规则是什么
- 保证金余额怎么看
- 冻结保证金是什么意思
- 申诉流程是什么
- 费率规则怎么查看

## 无效问题

无法识别为平台规则、经营数据或商家资料的问题，回复：`请提工单进行人工咨询`，且不写入 `merchant_ai_answer`。
